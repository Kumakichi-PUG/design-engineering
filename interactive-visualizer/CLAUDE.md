# Interactive Visualizer

## プロジェクト概要

マイク入力にリアルタイムで反応する、ベクターベースの2段階ビジュアライザー。Webブラウザ上で動作する。

## 基本仕様

### アートボード
- サイズ: 1920 x 1080（16:9）
- マージン: 上下左右 40px（額縁状）
- 描画領域: 1840 x 1000（マージン内側）
- 背景: #f8f8f8

### グラフィック方針
- グラデーション不使用
- ベクターデータのみ（SVG的表現）
- クリーンなパスで構成
- 配色: 黒（#1a1a1a）と白（#ffffff）の2色のみ

---

## ビジュアライザー構造

### 2つのオブジェクトの関係

黒オブジェクト（外側）の上に白オブジェクト（内側）が乗ることで、黒が**均一幅の額縁**として見える構造。

### Layer 1: 黒オブジェクト
- **形状**: F字型などの凹凸を持つ直角のみで構成されたクローズドパス
- **挙動**: 音量に応じて右方向に膨張・収縮（呼吸的）
- **ノッチの挙動**: 音量が大きいほど凹みのネガティブスペースが縮小する
  - 静音時: ノッチは元のサイズを維持
  - 大音量時: ノッチが閉じる方向に変形（最小10pxまで縮小可）
  - 結果: 音量が上がると黒い領域が増え、より塊感のある印象になる
- **分割**: 分割数・分割方法はパラメーター制御（整数均等 / ランダム）
- **位置**: 描画領域内（40pxマージンの内側）

### Layer 2: 白オブジェクト
- **形状**: 黒と同じ頂点数・同じトポロジーの相似形
- **全頂点が直角**で構成
- **オフセット**: 黒のクローズドパスの内側に全頂点が収まる（Illustratorの「パスのオフセット」的）
- **オフセット量**: 約60px（黒のどの辺からも垂直方向に60px内側）
- **凹部の扱い**: 黒の凹み（ノッチ）部分では、白の頂点は凹壁より内側（ステム側）に位置する。結果として白のノッチは黒のノッチより大きくなり、凹部の3辺すべてに黒フレームが見える
- **重要**: 白の頂点は1つも背景（#f8f8f8）の上に乗ってはならない。必ず黒の塗り領域の上にある

### 頂点の対応ルール
```
黒の頂点 1, 2, 3, 4, 5, 6, 7, 8
              ↕  ↕  ↕  ↕  ↕  ↕  ↕  ↕   （1:1対応）
白の頂点 1',2',3',4',5',6',7',8'
```
- 頂点数は黒と白で必ず同じ
- 頂点を増やしも減らしもしない

### レイヤー構造（断面図）
```
┌──────────────────────────────┐ ← #f8f8f8 背景
│                              │
│  ┏━━━━━━━━━━━━━━━━━━━━━━┓  │ ← 黒オブジェクト（外側）
│  ┃  ┌──────────────────┐ ┃  │
│  ┃  │                  │ ┃  │ ← 白オブジェクト（内側）
│  ┃  │                  │ ┃  │
│  ┃  └──────────────────┘ ┃  │
│  ┃   ← 60px 黒フレーム → ┃  │
│  ┗━━━━━━━━━━━━━━━━━━━━━━┛  │
│                              │
└──────────────────────────────┘
```

## ジグザグ化（音声反応）

### パス構造
1. 白の相似形（フラットな直角パス）をベースとする
2. その辺上に**等間隔でポイント**を配置
3. **奇数番ポイント**: オフセット線上に固定（動かない）
4. **偶数番ポイント**: 内側に引っ込む（音量に連動）

### 挙動
- 静音時: 偶数番ポイントもオフセット線上 → フラットな相似形
- 音量 ↑: 偶数番が内側に引っ込む → ジグザグ化（トゲが立つ）
- 音量 ↓: 偶数番がオフセット線に戻る → フラットに戻る
- 結果: 音量が大きいほどトゲが鋭く、攻撃的な印象になる

### アンビエントモーション（無音時の微動）
- 白の頂点は**無音環境でも常に微かに動く**
- 完全に静止せず「生きている」印象を与える
- 音量が入ると、このアンビエントモーションの上に音声反応が加算される

### パラメーター制御
- ポイント数（等間隔の密度）→ トゲの数を制御
- 偶数番の引っ込み量 → 音量マッピングの感度

---

## 形状パラメーター

### 凹凸の数（外形の複雑さ）
- 黒オブジェクトの外形が持つノッチ（切り欠き）の数をパラメーターで制御
- 値が小さい → シンプルな形状（矩形に近い）
- 値が大きい → 複雑なギザギザ形状
- 全てのノッチは直角で構成

### 頂点数の連動ルール
- **黒と白の頂点数は常に同じ**（1:1対応を維持）
- 黒の凹凸が増える → 黒の頂点数が増える → 白も同じ数に自動追従
- 黒の凹凸が減る → 黒の頂点数が減る → 白も同じ数に自動追従
- 白のオフセット量（60px）と直角ルールは頂点数が変わっても維持

## 音声入力
- ソース: マイク入力（Web Audio API / getUserMedia）
- リアルタイム処理
- FFT解析で音量・周波数を取得

## GUI
- スライダー形式のパラメーター制御UI
- 制御項目:
  - 凹凸の数（外形の複雑さ）
  - ブロック分割数
  - 分割方法（均等 / ランダム）
  - トゲ数（ポイント密度）
  - 音量感度

## 技術スタック
- **描画エンジン**: Paper.js
  - パスのオフセット（path.offset）で相似形生成
  - ブーリアン演算、交差判定、パス上の等間隔ポイント配置
  - Canvas描画で60fps
  - project.exportSVG() でSVGエクスポート
- **音声入力**: Web Audio API（getUserMedia + AnalyserNode）
- **GUI**: lil-gui またはHTML range input（スライダー）
- **ビルド**: TBD（Vite推奨）

## Cursor での作業方針
- このCLAUDE.mdを仕様書として参照しながら実装
- まず黒オブジェクトの静的描画 → 白オフセット → ジグザグ → 音声連動の順で段階的に進める
- 各段階でブラウザ確認（`npx vite`）
- Cursorのチャットにこのファイルを添付するか、プロジェクトルートに置いておけばコンテキストとして自動参照される
